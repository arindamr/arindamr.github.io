LLM-Powered Diagnostic Engine: Key Concepts

Introduction

  • Fully automated software issue triage using Large Language Models (LLMs) presents significant challenges, including the potential for hallucinations, biases, and difficulties in interpreting complex diagnostic outputs.
  • A human-in-the-loop approach is proposed to leverage the analytical power of LLMs while retaining the crucial oversight and domain expertise of human engineers.
  • This integrated solution aims to enhance the efficiency and accuracy of software issue triage by combining the strengths of AI and human intelligence.
  • The core principle is to have human review and approval at critical junctures in the automated diagnostic process.

Architecture Overview

  • Problem Statement Input: Software engineers provide an English language description of the issue.
  • LLM Core: This acts as the central orchestrator of the diagnostic process, analysing the problem statement and suggesting actions.
  • Tool Registry: A repository containing comprehensive information about available diagnostic tools, including their purpose, capabilities, and usage instructions.
  • Execution Engine: Responsible for executing the selected diagnostic tools with the provided parameters and returning their output.
  • Output Interpreter: A module that helps the LLM Core understand the results of the tool execution.
  • Iterative Cycle: The process involves continuous loops of LLM suggestion and human review/modification until the issue is understood and a resolution is identified.

Refining the Problem Statement

  • The process begins with the engineer providing a problem statement.
  • The LLM Core can analyse this statement and suggest enhancements for clarity and completeness, or extract key information.
  • Crucially, the enhanced problem statement is presented to a human engineer for review.
  • The engineer can then:
    • Assess the LLM's suggestions.
    • Make necessary modifications to correct misinterpretations.
    • Add crucial details missed by the LLM.
    • Rephrase the problem statement based on their initial understanding.
  • This human review ensures an accurate foundation for subsequent triage steps.

Selecting Diagnostic Tools

  • Based on the refined problem statement, the LLM Core interacts with the Tool Registry.
  • The LLM suggests the diagnostic tool it deems most appropriate for the situation.
  • However, the LLM does not automatically execute the tool; its suggestion is presented to a human engineer for approval.
  • The engineer can then:
    • Approve the LLM's suggestion.
    • Select an alternative tool from the Tool Registry based on their experience and knowledge.
  • This human intervention ensures that the chosen tool is the most suitable for diagnosing the issue.
